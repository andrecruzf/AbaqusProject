\chapter{Conclusion}\label{chap:conclusion}

This project developed a fully automated pipeline for the numerical determination of Forming Limit Curves using three test configurations — Nakazima, Marciniak, and Punch-in-Punch — in Abaqus/Explicit. The pipeline covers the complete workflow from model building and HPC job submission to strain extraction, quasi-static validation, and FLC plotting, and runs end-to-end on the ETH Euler cluster from a single deployment command.

The key contributions are:
\begin{itemize}
  \item A modular, configuration-file-driven pipeline (\texttt{config.py}) supporting all three test types, seven specimen widths, variable sheet thickness, and material orientation angle, with no code changes required between configurations.
  \item A correct implementation of the two-step Punch-in-Punch boundary conditions in Abaqus/Explicit: Punch~1 is held at its Step~1 end-position via an \texttt{EncastreBC} (zero nodal velocity) in Step~2, while Punch~2 advances a further 20\,mm.
  \item Forming speeds of 2\,mm/s throughout, compliant with the upper bound of ISO~12004-2 (0.5--2\,mm/s), with fixed mass scaling ($\Delta t = 2 \times 10^{-5}$\,s) verified to maintain ALLKE/ALLIE $< 5$\% across all widths.
  \item Automated failure detection restricted to the ISO-compliant dome zone ($R_\mathrm{dome} = 15$\,mm for the Nakazima/Marciniak punch, scaled appropriately for PiP), with base-fracture rejection.
  \item Per-simulation quasi-static validation plots (ALLKE/ALLIE vs.\ time) produced automatically alongside the strain path data.
\end{itemize}
