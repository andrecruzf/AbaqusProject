\chapter{Numerical Model}\label{chap:numerical_model}

\section{Model Overview}

The FE model replicates the selected forming test in Abaqus/Explicit, chosen via the \texttt{TEST\_TYPE} flag (\texttt{nakazima}, \texttt{marciniak}, or \texttt{pip}) in the central configuration file. Due to two planes of symmetry, only a quarter of the specimen is modelled, significantly reducing computational cost.

For the PiP configuration, the model includes five parts:
\begin{itemize}
  \item \textbf{Blank}: deformable solid, meshed with the supervisor-provided specimen geometry.
  \item \textbf{Punch~1}: outer annular rigid body (inner bore $r = 20$\,mm, outer $r = 43.75$\,mm).
  \item \textbf{Punch~2}: inner hemispherical rigid body ($R = 15$\,mm).
  \item \textbf{Die}: rigid ring with flat contact face and 15\,mm fillet.
  \item \textbf{Blank holder}: rigid body with 2\,mm inner chamfer, inner radius 62.5\,mm.
\end{itemize}

For Nakazima and Marciniak, the standard four-part model is used (single punch, die, blank holder, blank). Specimen geometry files (\texttt{.cae}) are provided for widths $W \in \{20, 50, 80, 90, 100, 120, 200\}$\,mm and are shared between all test configurations.

\section{Discretisation}

The blank is meshed with eight-node linear brick elements with reduced integration and hourglass control (\texttt{C3D8R}). A single element layer is used through the thickness, consistent with the supervisor-provided geometry files and with the mesh configuration used in the reference model (PinP\_CR210H). Element size in the dome zone is approximately 1\,mm, providing adequate resolution of the strain gradient without excessive computational cost for the explicit solver.

Maintaining a consistent element size across all specimen widths and test configurations is a primary requirement of the project outline (Task~III). All geometry files use a common mesh density in the contact zone, ensuring that the extracted FLC points are not influenced by spatial discretisation artefacts. A mesh sensitivity study — varying element size systematically and monitoring the extracted failure strains — is identified as a direction for future work (see Chapter~\ref{chap:future_work}).

\section{Material Model}

The blank material is defined via a VUMAT Fortran subroutine (\texttt{VUMAT\_explicit.f}), which implements the Hosford-Coulomb criterion with 46 material constants. The subroutine computes the stress update using a radial return algorithm and evaluates the fracture criterion at each increment. Initial conditions for SDV8 ($\beta = 1.0$) and SDV17 ($\bar{\varepsilon}^p_f$) are injected programmatically at the start of the step to work around an Abaqus 2023 CAE limitation with VUMAT predefined fields.

\section{Boundary Conditions and Interactions}

Symmetry boundary conditions are applied on the two cut planes of the quarter model. The blank holder and die are fully fixed. Surface-to-surface kinematic contact is defined between the blank and each rigid tool, with Coulomb friction coefficients set in \texttt{config.py}.

For the PiP configuration, the two-step process is controlled by separate boundary conditions on each punch reference point:
\begin{itemize}
  \item \textbf{Step~1} (\texttt{Step1\_Clamp}, 10\,s): both Punch~1 and Punch~2 advance 20\,mm together with a linear amplitude.
  \item \textbf{Step~2} (\texttt{Step2\_Form}, 10\,s): Punch~1 is locked via an \texttt{EncastreBC} (zero nodal velocity, holding at end-of-step-1 position); Punch~2 advances a further 20\,mm with a linear amplitude.
\end{itemize}

For Nakazima and Marciniak, a single \texttt{Forming} step (18.5\,s) uses a smooth-step amplitude to prescribe the 37\,mm punch displacement.

\section{Forming Speed and Mass Scaling}

All simulations use a punch speed of 2\,mm/s, consistent with the upper bound specified in ISO~12004-2~\cite{ISO120042} (0.5--2\,mm/s). This is an artificial speed used in the quasi-static Abaqus/Explicit framework: the step time is set to $t = d/v$ (18.5\,s for Nakazima/Marciniak; $2 \times 10$\,s for PiP).

Fixed mass scaling with a target stable time increment $\Delta t = 2 \times 10^{-5}$\,s is applied to maintain computational efficiency. The quasi-static assumption is validated post-hoc by verifying that the ratio of kinetic to internal energy (ALLKE/ALLIE) remains below 5\% throughout the simulation. The mass scaling value was chosen accounting for the reduced punch speed relative to earlier implementations: since ALLKE $\propto \Delta t^2 v^2$, reducing $v$ from 5\,mm/s to 2\,mm/s allows $\Delta t$ to be increased by a factor of 2.5 while maintaining the same energy ratio.

\subsection{Mass Scaling Sensitivity Study}

To validate the chosen increment $\Delta t = 2 \times 10^{-5}$\,s, a dedicated sensitivity study was performed using \texttt{deploy\_mass\_scaling.sh}. One representative geometry (Nakazima W100, $t = 1.85$\,mm) was simulated with four values: $\Delta t \in \{1\times10^{-5},\, 2\times10^{-5},\, 5\times10^{-5},\, 1\times10^{-4}\}$\,s. Each value produces an independent output directory (suffix \texttt{\_ms2e5}, etc.); after all solver jobs complete a comparison job runs \texttt{plot\_mass\_scaling.py} and writes a two-page PDF overlaying the ALLKE/ALLIE ratio and absolute energies for all DT values. The 5\,\% quasi-static threshold is drawn as a reference line on the ratio plot. The selected $\Delta t = 2 \times 10^{-5}$\,s was confirmed to keep the peak ratio well below this threshold while remaining computationally efficient.

\section{Output Requests}

Field outputs are requested at regular intervals and include:
\begin{itemize}
  \item \texttt{LE}: Logarithmic strain tensor (used to compute principal strains $\varepsilon_1$, $\varepsilon_2$).
  \item \texttt{SDV1--17}: All solution-dependent variables (EQPS, stress state, fracture strain).
  \item \texttt{STATUS}: Element status (0 = deleted, used to detect failure frame).
  \item \texttt{S}, \texttt{U}, \texttt{RF}: Stress, displacement, reaction forces.
\end{itemize}

History output includes whole-model energies \texttt{ALLKE} and \texttt{ALLIE} for quasi-static validation.

\section{Simulation Pipeline}

The entire workflow is automated via a set of Python and shell scripts deployed on the ETH Euler HPC cluster. The pipeline runs end-to-end from a single \texttt{deploy\_all.sh} invocation on the local machine.

\begin{enumerate}
  \item \texttt{config.py} — Central parameter file: test type, specimen width, thickness, friction coefficients, VUMAT constants, forming speed, output directory.
  \item \texttt{deploy\_all.sh} — Pushes all scripts to Euler via \texttt{scp}, then launches \texttt{submit\_all.sh} in a detached \texttt{tmux} session on the login node. The local terminal is freed immediately; all model building and job submission proceed remotely.
  \item \texttt{submit\_all.sh} (runs on Euler) — Loads the Abaqus module, loops over all specimen widths, calls \texttt{abaqus cae noGUI=build\_model.py} for each, and submits the corresponding solver job. After all solver jobs are queued, the FLC aggregation job is submitted with an \texttt{afterok} dependency on all solver IDs. Output is logged to \texttt{submit\_all.log}.
  \item \texttt{build\_model.py} — Calls 8 modules sequentially to build the Abaqus model (\texttt{parts}, \texttt{assembly}, \texttt{material}, \texttt{step}, \texttt{contact}, \texttt{boundary}, \texttt{output}, \texttt{job}) and write the \texttt{.inp} file.
  \item Abaqus/Explicit solver — Runs the simulation on the compute node (24 CPUs, \texttt{mp\_mode=threads}).
  \item \texttt{postproc.py} — Extracts principal strains and EQPS at failure $\to$ \texttt{strain\_path.csv}; plots ALLKE/ALLIE vs.\ time $\to$ \texttt{energy\_ratio.png}.
  \item \texttt{postproc\_movie.py} — Renders a SDV1 (EQPS) animation $\to$ \texttt{.webm}.
  \item \texttt{flc\_plot.py} — Aggregates all \texttt{strain\_path.csv} files across widths and plots the FLC $\to$ \texttt{flc\_diagram.png}.
\end{enumerate}

\section{Failure Detection}

The failure frame is identified as the last output frame before the first element deletion, detected by a drop in \texttt{STATUS} below 0.5. The search is restricted to the \textit{dome zone}: elements whose undeformed centroid lies within a radius $R_\mathrm{dome}$ of the punch axis ($X = Y = 0$), set to 15\% of the punch diameter per ISO~12004-2. If the first deletion occurs outside the dome zone, the fracture is classified as a \textit{base fracture} and the corresponding FLC point is excluded from the curve.

At the pre-failure frame, the element with the maximum SDV1 (EQPS) in the dome zone is identified as the critical element. Its major and minor principal strains, computed analytically from the LE tensor eigenvalues, define one point on the FLC.
